# Research: Framework para Descriptores de Cargo Formales — CognitionBase

## Vision

Determinar el framework conceptual correcto para formalizar descriptores de cargo en CognitionBase (spinoff AI-First de 23people.io), partiendo del borrador existente de Lead Dev AI. Producir distinción canónica entre Cargo, Rol, Función y Responsabilidad, un template aplicable, y un ejemplo concreto.

## Scope

- In scope:
    - Estructura canónica de un descriptor de cargo formal (mejores prácticas RRHH moderno)
    - Diferenciación conceptual: Cargo / Posición / Rol / Función / Responsabilidad
    - Modelado de cargos con múltiples roles simultáneos (ej: Lead Dev AI ejerce rol SDM)
    - Gestión de niveles de experiencia (seniority tracks) dentro de un cargo
    - Frameworks modernos: job architecture, competency frameworks, RACI — cuáles aplican en startups
    - Enfoque pragmático para empresas AI-First pequeñas
    - Ejemplo aplicado al caso Lead Dev AI de CognitionBase
- Out of scope:
    - Política de compensación o bandas salariales
    - Procesos de reclutamiento o selección
    - Performance management / evaluaciones de desempeño
    - Aspectos legales laborales chilenos en detalle
    - Cargos para agentes IA (confirmado fuera de scope)

## Success Criteria

- [ ] Framework conceptual claro: Cargo vs Rol vs Función vs Responsabilidad
- [ ] Template recomendado con todos los campos para descriptor de cargo formal
- [ ] Modelo documentado para cargos con múltiples roles asociados
- [ ] Estrategia para niveles de experiencia (seniority tracks)
- [ ] Recomendación sobre nomenclatura formal (Job Position vs Role vs Cargo)
- [ ] Ejemplo aplicado: descriptor Lead Dev AI de CognitionBase enriquecido
- [ ] Candidatos para notas Zettelkasten identificados

## Company Context

- CognitionBase: spinoff de innovación de 23people.io (staffing tech chilena)
- Cultura Output-First, AI-First, sin burocracia excesiva
- Iron-Legion: equipo mixto humanos + agentes IA (agentes son actores/herramientas, no tienen cargo)
- Roles operativos actuales: Lead Dev AI (que ejerce además rol SDM), SRM, Devs AI
