La carpeta `cognitionbase` contiene tres subcarpetas con roles distintos:

- **docs/**: Documentos resultantes de otras tareas que ya son permanentes y validados. Son los archivos oficiales y consolidados.
- **references/**: Archivos con información obtenida de fuentes externas (investigación, datos de referencia, etc.).
- **fleeting-tasks/**: Funciona como un inbox. Aquí deben quedar todos los documentos nuevos generados o modificados — son versiones pre-oficiales, posiblemente temporales, creados para apoyar otra tarea. Después se ordenan hacia `docs/` o `references/` según corresponda.

**Regla de uso:** Todo documento nuevo que se genere debe guardarse primero en `fleeting-tasks/`. Luego el usuario decidirá si promoverlo a `docs/` o `references/`.
